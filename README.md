# March Madness 2026

Tracks 1 billion generated March Madness brackets against real tournament results in real-time.

Built as a portfolio project. See [ARCHITECTURE.md](./ARCHITECTURE.md) for full system design.

## Concept

Every possible bracket can be deterministically generated from an integer index (like a Library of Babel for brackets). This project defines a space of 1 billion brackets, each generated by seeding a PRNG with the bracket's index. As real tournament results come in, we filter the space to find how many of our brackets still match reality.

The model uses real KenPom AdjEM (adjusted efficiency margin) ratings via a logistic function to generate more realistic brackets — not just random coin flips.

## Setup

### Requirements
- Node.js >= 20.9.0 (required by Next.js 16)
- npm

### Install
```bash
npm install
```

### First-time database setup
The SQLite database (`march-madness.db`) is created automatically on first run.
It will be seeded with all 63 tournament games when the first API call is made.

```bash
npm run dev

```

### Production (on server Mac)
```bash
npm run build
npm run start
# App runs on http://localhost:3000
```

### Hosting publicly with Cloudflare Tunnel
```bash
# One-time setup (install cloudflared):
brew install cloudflare/cloudflare/cloudflared

# Start tunnel (replaces port forwarding, no router config needed):
cloudflared tunnel --url http://localhost:3000
# Gives you a public HTTPS URL like https://abc-xyz-123.trycloudflare.com

# For a permanent URL, set up a named tunnel:
# https://developers.cloudflare.com/cloudflare-one/connections/connect-networks/get-started/
```

### Keep the server running (macOS)
```bash
# Install pm2 to auto-restart on crash and reboot:
npm install -g pm2
pm2 start "npm run start" --name march-madness
pm2 save
pm2 startup  # follow the instructions it prints
```

## Usage

1. Open the dashboard at `/`
2. Click **Refresh Analysis** to fetch latest ESPN scores and recompute bracket stats
   - First run with game results takes ~3 minutes (iterates 1 billion brackets)
   - Subsequent runs: same time (always iterates full space)
3. The dashboard shows:
   - How many of 1B brackets are still alive
   - Championship probability per team
   - Recent results and their bracket impact

## Key Files

| File | Purpose |
|------|---------|
| `lib/prng.ts` | mulberry32 PRNG — deterministic bracket generation |
| `lib/worker.ts` | Worker thread — parallel bracket analysis engine |
| `lib/tournament.ts` | Team data, win probabilities, bitmask computation |
| `lib/analyze.ts` | Orchestrates worker threads, aggregates stats |
| `lib/db.ts` | SQLite schema and queries |
| `lib/espn.ts` | ESPN API integration for live scores |
| `data/tournament-2026.json` | 64 teams with seeds and KenPom AdjEM ratings |

### The probability model
Win probability = `1 / (1 + exp(-0.07 * (netRatingA - netRatingB)))`
where `netRating` is the KenPom AdjEM as of Selection Sunday 2026.
Later-round probabilities are currently set to 0.5 (coin flip) — see TODO.md for V2 improvement.

### Performance
- Intel i9 8-core (2019 MacBook Pro): ~5-7M brackets/sec → ~150-200 seconds for 1B
- Apple M-series: ~10-13M brackets/sec → ~75-100 seconds for 1B
